1. FILE HANDLING

What Is File Handling?

File Handling is the process of creating, reading, updating and managing data inside files.

Why Is It Important?

Because real-world applications need permanent storage.

Real-Life Applications

✓ Student Management Systems

✓ Employee Databases

✓ Hospital Records

✓ Banking Systems

✓ Contact Books

✓ Project Reports

Benefits

✓ Permanent Data Storage

✓ Easy Record Management

✓ Professional Applications

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2. OPENING A FILE

Why?

Before working with a file, Python must open it.

Syntax

open(filename, mode)

Example

open("student.txt", "w")

Understanding

student.txt = File Name

w = File Mode

Benefits

✓ Access File Data

✓ Perform Operations

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3. WRITE MODE

Mode

w = Write

Purpose

Create a file and store data.

Example

file = open("student.txt", "w")

Important Rule

If the file already exists,

old content is replaced.

Benefits

✓ Save New Data

✓ Create Reports

✓ Store User Information

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4. READ MODE

Mode

r = Read

Purpose

Read data from an existing file.

Example

file = open("student.txt", "r")

Important Rule

Read Mode cannot modify data.

Benefits

✓ View Records

✓ Display Information

✓ Search Content

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5. APPEND MODE

Mode

a = Append

Purpose

Add new information without deleting old content.

Example

file = open("student.txt", "a")

Difference

Write Mode

↓

Deletes Old Data

Append Mode

↓

Keeps Old Data

↓

Adds New Data

Benefits

✓ Update Records

✓ Preserve Existing Data

✓ Maintain Logs

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

9. TRY BLOCK

Purpose

Contains risky code that may generate errors.

Syntax

try:
    risky_code

Meaning

Python says:

"I will try to execute this code."

Benefits

✓ Controlled Execution

✓ Safe Error Detection

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

10. EXCEPT BLOCK

Purpose

Handle errors safely.

Example

except:
    print("Invalid Input")

Benefits

✓ Prevents Crashes

✓ Shows Friendly Messages

✓ Improves Reliability

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

11. FINALLY BLOCK

Purpose

Execute code regardless of errors.

Important Rule

finally always executes.

Whether

✓ Error Occurs

OR

✓ No Error Occurs

Example

finally:
    print("Program Finished")

Benefits

✓ Cleanup Tasks

✓ Resource Management

✓ Guaranteed Execution

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

12. VALUEERROR

What Is ValueError?

Occurs when data conversion fails.

Example

int("SOHAN")

Reason

Text cannot be converted into an integer.

Benefits Of Handling It

✓ Better Validation

✓ User-Friendly Programs

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

13. ZERODIVISIONERROR

What Is ZeroDivisionError?

Occurs when a number is divided by zero.

Example

100 / 0

Reason

Division by zero is mathematically undefined.

Benefits Of Handling It

✓ Prevent Program Failure

✓ Better Stability
